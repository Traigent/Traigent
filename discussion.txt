KEY TECHNICAL DISCOVERIES
-------------------------

1. TRAIGENT INPUT EXPANSION
   - TraiGent expands the "input" dict as kwargs to the decorated function
   - Only fields inside "input" are passed to the function
   - Fields outside "input" and "output" go into example.metadata
   - Extra fields in "input" will cause TypeError if function doesn't accept them

2. EVALUATOR VS CUSTOM_EVALUATOR
   - evaluator= at decorator level: accepts function returning float (0.0-1.0)
   - custom_evaluator in EvaluationOptions: requires ExampleResult return type
   - Use evaluator= for simple scoring functions

3. COST ENFORCEMENT
   - Decorator parameter: cost_limit=10.00
   - Environment variable: TRAIGENT_RUN_COST_LIMIT (takes precedence)
   - Default limit: $2.00
   - To run expensive optimizations, need BOTH:
     * TRAIGENT_RUN_COST_LIMIT=10.00
     * TRAIGENT_COST_APPROVED=true

4. COST ESTIMATE FOR MIXED DATASET
   - 142 questions x 5 trials x 2 reps = 1,420 API calls
   - Estimated cost: $8.52
   - Command to run:
     source .venv/bin/activate && TRAIGENT_RUN_COST_LIMIT=10.00 TRAIGENT_COST_APPROVED=true python examples/quickstart/01_simple_qa_real_llm.py
