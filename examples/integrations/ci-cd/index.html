CI/CD Quality Gates

Advanced 20 min Automation

Goals

Integrate Traigent optimization gates into your CI/CD pipeline using only the SDK's native capabilities - no custom evaluation code required.

  • Regression Gate: Prevent merging if configuration underperforms baseline
  • Improvement Gate: Alert when auto-tuning finds better configurations
  • Zero Custom Code: Use only Traigent decorator and SDK methods
  • Specification-Driven: Configure behavior through environment variables

Structure

integrations/ci-cd/ math-qa/ # Example: Arithmetic Q&A math_qa.py # Decorated function math_qa.jsonl # Dataset (expressions -> answers) saved_config.json # Current production config run.py # CI runner (MODE=eval|tune) hooks/ # Git hooks pre-commit # Quick validation pre-push # Optimization check README.md # Full documentation

Quick Start

Test the CI/CD integration locally:

cd examples/integrations/ci-cd/math-qa # Evaluate current configuration TRAIGENT_MOCK_LLM=true MODE=eval python run.py # Run optimization to find better config TRAIGENT_MOCK_LLM=true MODE=tune MAX_TRIALS=10 python run.py

Environment Variables

  • MODE: eval (evaluate config) or tune (optimize)
  • MAX_TRIALS: Maximum optimization trials (default: 10)
  • TRAIGENT_MOCK_LLM: Use mock LLM for testing
  • REGRESSION_THRESHOLD: Max allowed degradation (default: -0.01 = 1%)
  • IMPROVEMENT_MIN_PCT: Min improvement to trigger alert (default: 0.03 = 3%)

CI Gates Logic

Gate 1: Regression Detection - If current performance is worse than baseline by more than the threshold, CI fails.

Gate 2: Missed Improvement - If tuning finds significantly better configurations, warn or fail so improvements aren't lost.
