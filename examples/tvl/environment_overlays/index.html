TVL Example

Environment Overlays

Advanced 15 min Multi-environment deployment

What You'll Learn

This advanced example demonstrates how to manage multiple deployment environments using TVL's extends/overrides pattern for staging and production configurations.

  • Define environment-specific overrides for configuration space
  • Use extends for inheritance between environments
  • Set environment-specific objective thresholds
  • Configure different promotion gates per environment
  • Integrate with TraiGent deployment targets

Environment Structure

environments: staging: overrides: configuration_space: model: values: ["gpt-4o-mini", "claude-3-haiku"] temperature: range: [0.2, 0.6] optimization: budget: max_trials: value: 30 unit: count production: extends: staging # Inherit from staging overrides: configuration_space: max_tokens: range: [256, 768] objectives: - name: latency threshold: value: 750 unit: milliseconds promotion: gate: strict_dominance

Deployment Integration

# TraiGent deployment configuration triagent: pipeline: SupportChat deployment_targets: ["staging", "production"] # This enables: # 1. Automatic environment detection # 2. Configuration resolution based on target # 3. Promotion gate enforcement per environment

Running This Example

# Validate operational configuration tvl-check-operational examples/tvl/environment_overlays/overlays.tvl.yml # Check for specific environment tvl-check-operational examples/tvl/environment_overlays/overlays.tvl.yml --env staging tvl-check-operational examples/tvl/environment_overlays/overlays.tvl.yml --env production
