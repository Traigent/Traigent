Constraints & Units

Intermediate 10 min CEL expressions and type safety

What You'll Learn

This example explores the powerful constraint system in TVL, including CEL expressions and unit typing for type-safe optimization.

  • expression constraints for simple rules
  • conditional constraints with when-then patterns
  • forbidden constraints for explicit prohibitions
  • Unit typing for dimensional analysis (ratio, tokens, hours)
  • Compound expressions for complex validation

Constraint Types

constraints: # Expression constraint: simple rule - id: cache-ttl-unit-check type: expression rule: 'params.cache_ttl_hours >= 1.0' error_message: "Cache needs at least 1 hour TTL." # Expression constraint: compound expression - id: token-temperature-budget type: expression rule: 'params.temperature * params.max_tokens <= 600' error_message: "Token-temperature product keeps hallucinations manageable." # Conditional constraint: when-then pattern - id: rerank-mandate type: conditional when: 'params.max_tokens >= 512' then: 'params.rerank_weight >= 0.4' error_message: "Long answers must lean on reranking." # Forbidden constraint: explicit prohibition - id: rerank-cap type: forbidden rule: 'params.rerank_weight > 0.8' error_message: "Rerank weight above 0.8 is unstable."

Unit Definitions

units: time: [milliseconds, seconds] currency: [USD] ratio: [dimensionless] percentage: [percent] configuration_space: temperature: type: continuous range: [0.0, 1.0] unit: ratio # Dimensionless ratio cache_ttl_hours: type: continuous range: [0.5, 12.0] unit: hours # Time unit

# Check structural validity and constraint syntax tvl-check-structural examples/tvl/constraints_units/constraints.tvl.yml # Full validation with tvl-lint tvl-lint examples/tvl/constraints_units/constraints.tvl.yml
