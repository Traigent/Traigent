## VS Code Test Discovery Fixed

### Problem
- Test discovery failed with collection error
- `test_optimization_validation.py` failed to import (decorator validates dataset at import time)

### Solution

1. Updated `.vscode/settings.json`:
```json
{
    "python.testing.pytestEnabled": true,
    "python.testing.pytestArgs": ["tests"],
    "python.testing.unittestEnabled": false,
    "python.envFile": "${workspaceFolder}/.env"
}
```

2. Created `.env` file:
```
TRAIGENT_MOCK_MODE=true
```

3. Fixed `tests/integration/test_optimization_validation.py`:
   - Added module-level skip if required datasets don't exist
   - Prevents import-time validation errors

After fix: 9682 tests collected, 1 skipped (no errors)

## New RAG Example Created

Created: `examples/quickstart/02_customer_support_rag_real_llm.py`

Based on `01_simple_qa_real_llm.py` pattern, adapted for RAG use case.

### Features
- Real OpenAI API calls via LangChain
- RAG retrieval depth optimization (k parameter)
- Model and temperature tuning
- Multi-objective optimization (accuracy, cost, latency)
- Custom `rag_accuracy_scorer` for response quality
- Constraints: GPT-4o low temp, GPT-3.5 needs more context
- CSV results export
- Summary by model and retrieval depth

### Configuration Space
```python
CONFIGURATION_SPACE = {
    "model": ["gpt-3.5-turbo", "gpt-4o-mini", "gpt-4o"],
    "temperature": [0.1, 0.3, 0.5, 0.7],
    "k": [2, 3, 5],  # RAG retrieval depth
}
```

### Run Command
```bash
export OPENAI_API_KEY="your-key-here"
python examples/quickstart/02_customer_support_rag_real_llm.py
```
