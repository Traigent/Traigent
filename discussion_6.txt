### Architecture
The test suite uses a declarative, scenario-based design:

```
tests/optimizer_validation/
├── conftest.py              # Core fixtures (scenario_runner, result_validator)
├── specs/                   # Test specification framework
│   ├── scenario.py          # TestScenario dataclass
│   ├── builders.py          # Factory functions
│   └── validators.py        # Result validation
├── dimensions/              # Feature tests (~70 tests)
├── failures/                # Error handling (~30 tests)
└── interactions/            # Combination tests (~18 tests)
```

### Test Categories

1. **Dimensions (Feature Tests)**
   - test_config_spaces.py (12) - Categorical, continuous, mixed configs
   - test_injection_modes.py (12) - context, parameter, attribute, seamless
   - test_execution_modes.py (10) - edge_analytics, privacy, hybrid, standard, cloud
   - test_objectives.py (14) - Single, multi-objective, weighted
   - test_constraints.py (10) - Config-only, metric-dependent, compound
   - test_evaluators.py (11) - Default, custom, scoring functions

2. **Failures (Error Handling)**
   - test_dataset_issues.py (12) - Empty, malformed, missing data
   - test_function_bugs.py (11) - Exceptions, invalid returns, timeouts
   - test_evaluator_bugs.py (~5) - Evaluator failures
   - test_invocation_failures.py (~5) - Invalid configs

3. **Interactions (Combinations)**
   - test_key_combinations.py (18) - Strategic feature combinations

## Known Issue After Merge

**1 test failing**: `test_simple_config_constraint`

The constraint test expects configs with temperature >= 0.8 to be filtered out,
but a config with temperature=0.9 is passing through.

```
AssertionError: assert 0.9 < 0.8
```

This appears to be a bug where constraints aren't being enforced during
config selection in the optimizer. The test correctly catches this behavior.

Test results after merge: 117 passed, 1 failed
