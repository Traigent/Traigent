## Part 4: Wiring Up Anchor-Guided Selection

### Changes to `create_dataset.py`:

**Before (random sampling):**
```python
examples = dataset.examples
rng = random.Random(random_seed)
shuffled = list(examples)
rng.shuffle(shuffled)
selected = shuffled[:size]
```

**After (anchor-guided TF-IDF selection):**
```python
from .eval_set_generator import AnchorGuidedSelector

anchor_set, candidate_pool, _ = dataset.get_stratified_split(
    anchor_size=anchor_size,      # 50
    candidate_size=candidate_size, # 400
    test_size=0,
)

selector = AnchorGuidedSelector(
    anchor_set=anchor_set,
    candidate_pool=candidate_pool,
    random_seed=random_seed,
)
result = selector.select_greedy(target_size=size)
selected = result.selected_examples
```

### Selection Algorithm Details:
- **Separation score**: `1 - max_similarity` to already-selected (diversity)
- **Alignment score**: Average cosine similarity to anchor set (representativeness)
- **Combined**: `0.6 * separation + 0.4 * alignment`
- **TF-IDF config**: max_features=1000, ngram_range=(1,2), stop_words="english"

## Part 6: TF-IDF vs Embeddings Discussion

### User noticed:
> "Wait, we are using TF-IDF rather than embedding space?"
> "But didn't I say embeddings?"

### Current implementation:
- Uses `sklearn.feature_extraction.text.TfidfVectorizer`
- TF-IDF = sparse, lexical overlap based
- NOT neural embeddings (dense, semantic similarity)

### Discrepancy:
- Docstrings say "embedding space" but code uses TF-IDF
- User's original intent was likely neural embeddings
- TF-IDF was already implemented, just not wired up

### Implications:
- TF-IDF captures lexical/syntactic patterns
- Neural embeddings would capture semantic similarity
- TF-IDF already gave significant improvement (63% → 80%)
- Upgrading to embeddings could potentially improve further
