# Pre-Launch Content Audit (Blocking)

Use this before publishing Week 1 content (and whenever you reuse older drafts).

## 1) Product/availability truth check

- [ ] Clearly state what’s available **today** (OSS SDK + `edge_analytics` + mock mode).
- [ ] Any mention of cloud/hybrid/managed services is labeled **roadmap** unless provisioned.

## 2) Claim hygiene

- [ ] No budget-independent claims (“tests thousands”, “hours not weeks”, “zero effort”).
- [ ] Integration claims are qualified (supported patterns/frameworks vs “may require adapters”).
- [ ] Any numeric threshold is scoped (“depends on stakes/domain”) or removed.

## 3) Links + assets

- [ ] Repo URL correct and reachable.
- [ ] Every post/article has exactly **one primary CTA**.
- [ ] Visual selected and attached (or a placeholder is present for Medium publishing workflow).

## 4) Code credibility

- [ ] Code snippets are runnable **or** clearly labeled “pseudo-code” with a link to a runnable example.
- [ ] If referencing a repo path, it exists and has a stable name.
