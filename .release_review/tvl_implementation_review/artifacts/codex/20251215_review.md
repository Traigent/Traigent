## Issues Found

### [P1] Chance constraint lower bound uses inexact Wilson interval

**File**: `traigent/tvl/statistics.py:93-161`

**Severity**: High (P1)

**Description**:
The `clopper_pearson_lower_bound` function is documented as an exact Clopper-Pearson binomial bound, but for `trials >= 30` it returns a Wilson score approximation instead of the exact beta quantile.

**Impact**:
Wilson lower bounds are generally higher than the exact CP bound, so near-threshold metrics can be marked as satisfying a chance constraint when the exact CP bound would fail, leading the promotion gate to approve unsafe candidates. This affects any environment using chance_constraints with moderate sample sizes.

**Recommendation**:
Either rename the function to indicate it's an approximation, or implement the true Clopper-Pearson bound using the beta quantile for all sample sizes.

---

### [P2] Paired comparison ignores pairing and length mismatch

**File**: `traigent/tvl/statistics.py:355-434`

**Severity**: Medium (P2)

**Description**:
The `paired_comparison_test` advertises a paired t-test and even documents raising on mismatched lengths, but the implementation performs an unpaired Welch test on the two sample means and never checks that `x_samples` and `y_samples` are the same length or aligned.

**Impact**:
This both allows unequal sample counts to pass silently and discards pairing information, producing different p-values/effect sizes than expected when metrics are collected per-request (incumbent vs candidate) in promotion gating.

**Recommendation**:
Either:
1. Implement a true paired t-test that requires equal lengths and uses the paired differences, or
2. Rename the function to indicate it's an unpaired comparison and update documentation accordingly.

## Resolution

### P1 Fix Applied
- Updated docstring to accurately describe the hybrid approach:
  - Small samples (n < 30): Uses exact Clopper-Pearson via beta quantile
  - Large samples (n >= 30): Uses Wilson score interval for computational stability
- Added note that Wilson may be slightly less conservative for borderline cases
- Function name kept as `clopper_pearson_lower_bound` for backward compatibility

### P2 Fix Applied
- Reimplemented `paired_comparison_test` as a true paired t-test:
  - Now computes paired differences (x_i - y_i)
  - Tests the mean of differences against epsilon
  - **Requires equal-length samples** - raises `ValueError` if lengths differ
  - Uses paired degrees of freedom (n - 1) instead of Welch-Satterthwaite
- Added test `test_mismatched_lengths_raises` to verify length validation
