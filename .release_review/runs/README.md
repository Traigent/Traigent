# Release Review Run Workspace

Each run uses:

`.release_review/runs/<release_id>/`

Required files/directories:

- `run_manifest.json`
- `REVIEW_LOG.md`
- `gate_results/check_results.json`
- `gate_results/verdict.json`
- `inventories/src_files.txt`
- `inventories/tests_files.txt`
- `inventories/review_scope_files.txt`
- `file_reviews/` (per file + role + agent artifacts)
- `waivers/`

Artifact quality expectations:
- component evidence includes `review_summary`, `checks_performed[]`, and `strengths[]`
- per-file approved artifacts include `checks_performed[]`, `strengths[]`, and explanatory `notes` when no defects are found

These files are generated by:

- `python3 .release_review/automation/generate_tracking.py`
- `python3 .release_review/automation/release_gate_runner.py`
- `python3 .release_review/automation/build_release_verdict.py`
